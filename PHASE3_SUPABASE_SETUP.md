# Phase 3: Supabase Setup Guide

## Prerequisites
- Supabase account created at https://supabase.com
- Project already initialized (credentials in `.env.local`)

## Setup Steps

### 1. Create the Locations Table
Go to Supabase Dashboard → SQL Editor and run the migration:
```sql
CREATE TABLE IF NOT EXISTS locations (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  name VARCHAR(255) NOT NULL,
  county VARCHAR(100) NOT NULL,
  lat DECIMAL(10, 8) NOT NULL,
  lng DECIMAL(11, 8) NOT NULL,
  type VARCHAR(50) NOT NULL,
  risk VARCHAR(20) NOT NULL,
  description TEXT,
  access TEXT,
  uploaded_by VARCHAR(100),
  date DATE DEFAULT CURRENT_DATE,
  likes INTEGER DEFAULT 0,
  photo TEXT,
  tags TEXT[],
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Indexes for performance
CREATE INDEX idx_locations_type ON locations(type);
CREATE INDEX idx_locations_risk ON locations(risk);
CREATE INDEX idx_locations_county ON locations(county);
CREATE INDEX idx_locations_name ON locations(name);
CREATE INDEX idx_locations_date ON locations(date DESC);

-- Row Level Security
ALTER TABLE locations ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow public read" ON locations FOR SELECT USING (true);
CREATE POLICY "Allow authenticated insert" ON locations FOR INSERT WITH CHECK (auth.role() = 'authenticated');
CREATE POLICY "Allow update likes" ON locations FOR UPDATE USING (true) WITH CHECK (true);
```

### 3. Test the Connection
Run the dev server and check the browser console for any errors:
```bash
npm run dev
```

You should see locations load from Supabase on page load.

## Key Changes Made
- `useLocations()` hook fetches data from `locations` table
- `handleUpload()` now inserts new locations via Supabase
- `handleLike()` updates like count in database
- Loading screen shows while fetching locations

## Column Mapping
- `id` - Auto-generated by Supabase (bigint)
- `name`, `county`, `lat`, `lng` - Location metadata
- `type` - One of: asylum, industrial, entertainment, mansion, hotel, church
- `risk` - One of: low, medium, high
- `description`, `access` - User-provided info
- `uploaded_by` - Username of uploader
- `date` - Upload date (YYYY-MM-DD)
- `likes` - Like counter
- `photo` - Photo URL (null for Phase 5)
- `tags` - Array of tag strings
- `created_at`, `updated_at` - Timestamps
